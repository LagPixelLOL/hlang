# Checklist: the HolyC language doc (`holyc-docs/README.md`)

Entries follow the doc's own order. "Tested" names files under `tests/`
unless marked *probe* (verified by running hcc during the audit, with
the reason it is not a golden).

| # | doc claim | implemented | tested | deviations |
|---|---|---|---|---|
| 1 | Types `U0 I0 I8 U8 I16 U16 I32 U32 I64 U64 F64`; **no F32** | yes | `cases/widening`, `edge/edge_arith`, `edge/edge_numbers`, `cases/misc` (`sizeof(I0)`/`sizeof(U0)` = 0) | `F32` is rejected, but via a generic parse error (it is just an unknown identifier), not a "no F32" message |
| 2 | Functions with no/default args callable **without parentheses** (`Dir;`) | yes | `cases/defaults`, `edge/edge_functions` (`T;`, `AllDflt+1`, `if (Toggle)`) | none |
| 3 | Default args **don't have to be on the end**; `Test(,3)` skips slots | yes | `cases/defaults` (doc example verbatim), `edge/edge_functions`, `errors/missing_arg`, `anti_c/anti_c_dft_callsite` | defaults are call-site expressions re-evaluated per call — doc-silent but TempleOS-faithful (`GrLine(CDC *dc=gr_dc,...)`); DEVIATIONS.md judgment calls |
| 4 | Lone char const → `PutChars()`; string → `Print()`; empty string `""` → variable fmt follows | yes | `cases/hello` (full `DemoHolyC` pattern incl. `'' drv;`) | none |
| 5 | `&name` for function addresses/callbacks | yes | `cases/classes`, `edge/edge_functions` (fn-ptr arrays, `Call()`) | hcc additionally allows calling a fn-ptr *variable* as `fp(...)` — the doc only shows `(*fp)(...)`/`Call()`; DEVIATIONS.md documents |
| 6 | **Postfix** type casts; `ToI64()`/`ToBool()`/`ToF64()`; "TempleOS follows normal C float<-->int conversion" | yes | `cases/pointers` (`3.9(I64)`, `ToI64`, `ToBool`, `ToF64`), `edge/edge_classes` (`&obj(A *)`) | F64→I64 **truncates**, matching the doc's wording; real TempleOS is widely reported to round-to-nearest (CVTSD2SI). Doc wording wins; ambiguity documented in DEVIATIONS.md |
| 7 | **No `main()`**; top-level code runs at startup, in order | yes | every golden test | AOT emits a `main()` wrapper that calls `__HC_startup` (DEVIATIONS.md) |
| 8 | No bit fields; byte/word access into any int; typed unions (`public I64i union I64 {...}`) | yes | `cases/subint` (doc's `i.u16[1]=0x9ABC` verbatim), `edge/edge_classes` (`r.raw.u8[0]`) | sub-int access is intrinsic on all int types; the KernelA.HH common-union declarations are unnecessary (user-defined ones work, tested) |
| 9 | Varargs `(...)` with `I64 argc`/`I64 argv[]`; `StrPrintJoin(NULL,fmt,argc,argv)` forwarding | yes | `cases/varargs` (`AddNums` doc example), `edge/edge_functions` | the doc's `GrPrint`/`GrPutS` graphics context is out of scope (DEVIATIONS.md) |
| 10 | Chained comparisons: `5<i<j+1<20`; `if (13<=age<20)` | yes | `cases/precedence` (doc example verbatim), `edge/edge_precedence`, `cases/floats` | middle terms evaluated **once**, short-circuit — hcc's reading of the doc's `&&` rewrite (which would evaluate `j+1` twice); DEVIATIONS.md documents |
| 11 | `switch []` skips the range check, "a little faster" | partial | `cases/switch` (accepted, correct results) | lowered through LLVM's switch → still bounds-checked; same or better speed, never wrong, just not literally unchecked (DEVIATIONS.md) |
| 12 | `switch` always uses a **jump table** | behavioral equivalent | `cases/switch`, `edge/edge_switch` | lowering strategy is LLVM's choice; observable behavior identical (DEVIATIONS.md) |
| 13 | Ranges: `case 4...7:` | yes | `cases/switch`, `cases/subswitch`, `edge/edge_switch` | none |
| 14 | Null case = one plus prev case, starting at 0 | yes | `cases/switch` (NullCase.HC verbatim), `edge/edge_switch` (range→null, interleaved) | none |
| 15 | `sub_switch`: `start:`/`end:` group cases with shared porches | yes | `cases/subswitch` (SubSwitch.HC verbatim + porch/`break` routing), `edge/edge_switch` (nested) | doc's "don't goto/throw/return out of the start porch" is advice, not diagnosed — same as TempleOS |
| 16 | `no_warn` suppresses unused-var warning | accepted | `cases/misc` | hcc emits no unused-var warnings at all, so `no_warn` parses to a no-op (DEVIATIONS.md) |
| 17 | Multiple members named `pad`/`reserved` allowed without warnings | yes | `anti_c/anti_c_class_pack` (two `pad` members) | enforced as the *only* repeatable names — other dup members rejected per scoping table (`errors/dup_member`) |
| 18 | `reg`/`noreg` before local names; optional named reg (`reg R15`) | parsed | `cases/misc` | register allocation is LLVM's job; `reg R15` pinning parses and is ignored (DEVIATIONS.md; asm interop is out of scope) |
| 19 | Function flags: `interrupt`, `haserrcode`, `public`, `argpop`, `noargpop` | parsed | `cases/misc` | `public`/`static` affect linkage; the rest are hosted no-ops (no IRQs/err codes) — DEVIATIONS.md |
| 20 | Single quotes hold **multiple chars**: `'ABC'==0x434241`; `PutChars()` takes packed chars | yes | `cases/hello`, `cases/widening`, `edge/edge_numbers` (8-char max), `edge/edge_strings` | >8 chars rejected (`edge/br_char_too_long`) |
| 21 | `` ` `` raises a base to a power | yes | `cases/precedence` (int, chained), `cases/floats` (F64, negative exponent) | left-associativity and unary-vs-`` ` `` precedence are doc-silent; the latter is deliberately not asserted (`edge/edge_precedence` explains) |
| 22 | **No question-colon operator** | yes (rejected) | `errors/ternary` | diagnostic quotes the doc: "there is no question-colon operator in HolyC -- use if/else" |
| 23 | TempleOS operator precedence table (`` ` ``,`>>`,`<<` / `*`,`/`,`%` / `&` / `^` / `\|` / `+`,`-` / cmp / `==`,`!=` / `&&` / `^^` / `\|\|` / assign) | yes | `cases/precedence` (`2+1&3==3`, `1<<4/2==8`, `4\|1^1`, `^^`...), `edge/edge_precedence`, `frontend/ast` | none |
| 24 | `Option(OPTf_WARN_PAREN,ON)` / `OPTf_WARN_DUP_TYPES` find lint issues | accepted | `cases/misc`, `cases/arrays` | the warnings themselves are not implemented; `Option()` is an accepted no-op (DEVIATIONS.md) |
| 25 | `#exe{}` runs code at compile time; `StreamPrint()` splices source | yes | `cases/exe` (values, statements, `#define` injection), `examples/CompileTime.HC` | runs inside hcc's runtime, not a TempleOS task: `Fs->last_cc` tricks and `Cd()`/`StreamDir` unavailable (DEVIATIONS.md) |
| 26 | **No `#define` functions** ("I'm not a fan") | yes (rejected) | `errors/define_func` | none |
| 27 | **No `typedef`**, use `class` | yes (rejected) | `errors/typedef` | none |
| 28 | **No type-checking** | yes | pervasive — e.g. `Bt(U8*...)` in `cases/memory`, `U0*`/`U8*` mixing in `cases/system` | hcc keeps a few targeted diagnostics (unknown member, undefined fun, too many args) — cases where TempleOS also errors |
| 29 | `#include ""` only, no `<>` | yes | `cases/preproc`, `errors/include_angle` | none |
| 30 | `$` is the DolDoc escape (`$$` = literal `$`); `$` also = inst address / class-offset | partial | `cases/strings`, `edge/edge_strings` (`$$`), `errors/dollar` | string escape fully handled; `$` as instruction address / class-offset hack **rejected** with a clear error (DEVIATIONS.md — no assembler) |
| 31 | `union` is class-like (no `union` label at use); type in front = whole-access type | yes | `cases/classes` (plain union), `cases/subint` (`public I64i union MyI64`), `edge/edge_classes` (punning) | none |
| 32 | Member **meta data** (`format`/`data`); all compiler structures saved; `DocForm()` etc. | partial | `cases/classes` (meta data parses and is ignored) | reflection not exposed: no `CHashClass`/`CMemberLst`/`MemberMetaData`/`ClassRep`/`DocForm`, no `Fs->hash_table` (DEVIATIONS.md) |
| 33 | `lastclass` keyword as default arg = prev arg's class name | yes | `cases/classes` (I64/F64/classes/`Fs`), `errors/lastclass_misuse` | its doc consumers (`ClassRep`, `DocForm`, BlkDevRep) are out of scope |
| 34 | `try{}`/`catch{}`/`throw` unlike C++; `throw` = function with ≤8-byte char arg; `Fs->except_ch`; `Fs->catch_except=TRUE` stops the search; `PutExcept()` as handler | yes | `cases/exceptions` (incl. `PutExcept`, bare `throw`, nested rethrow-outward), `examples/Oracle.HC`; doc's `Exceptions.HC` runs verbatim (*probe*: its `%P` prints live addresses → not goldenable) | setjmp-frame lowering; returning out of a `try{}` skips the frame pop — DEVIATIONS.md, mirroring TempleOS's own "don't return out of a catch" warning |
| 35 | `offset(classname.membername)`; `sizeof`/`offset` accept **one level only** | yes | `cases/classes`; two-level forms rejected (*probe*: generic parse error) | rejection is a generic `expected ')'` diagnostic rather than a "one level only" message |
| 36 | **No `continue`** — use `goto` | yes (rejected) | `errors/continue`, idiom tested in `cases/control`, `edge/edge_control` | diagnostic speaks HolyC ("use goto") |
| 37 | `lock{}` applies LOCK prefixes; "can be used without {}" | parsed+compiled | `cases/misc` (both brace and brace-less forms) | body compiles normally, no LOCK prefixes — hosted runtime is single-threaded, `JobQue`/`Spawn` absent (DEVIATIONS.md) |
| 38 | `MSize()` gives the real size of a heap object (heap rounds up) | yes | `cases/memory`, `cases/strings` | hcrt does **not** round allocations up to powers of two, so `MSize()` returns exactly the requested size — no slack to exploit |
| 39 | `Free(NULL)` is legal; `MAlloc` variants; other tasks' heaps; `HeapCtrlInit()` | partial | `cases/memory` (`Free(NULL)`, `MAlloc`/`CAlloc`/`MAllocIdent`) | per-task heaps and `HeapCtrlInit` out of scope (DEVIATIONS.md) |
| 40 | Stack does not grow; `MEM_DFT_STK`; `Spawn()`; `CallStkGrow()` | no (documented) | `edge/edge_recursion` exercises the *opposite* (50k frames just work) | host stacks grow; `CallStkGrow`/`Spawn` do not exist (DEVIATIONS.md) |
| 41 | **Only one base class** | yes | `cases/classes`, `edge/edge_classes` (4-deep chain), `errors/multi_base` | none |
| 42 | `printf()` has new codes — see `Print("")` fmt strings | yes (core set) | `cases/strings`, `edge/edge_strings` (flags, width, precision, `%z`, `%Q`, `%P`, `%,d`, packed `%c`) | `%3c` repeat counts and DolDoc codes missing; `%P` prints hex without symbol lookup (DEVIATIONS.md); `%u`/lowercase-`%x` are supersets not verifiable against the archive (Print.DD not preserved) |
| 43 | **All values extend to 64-bit on access; intermediates are 64-bit** — plus the 5-case example block | yes | `cases/widening` (doc block **verbatim**, incl. `i4>>1` vs `i5>>1` reg-var results), `edge/edge_arith` | none |
